Understanding Bonus Questionnaire Fields
Department

Indicates which department this question applies to. Each department can have its own set of tailored evaluation questions.

Example: Engineering, Sales, HR, Production
Question Text

The actual performance question that supervisors will see when evaluating employees. Questions should be clear, specific, and focused on measurable aspects of performance.

Examples:
  • "How effective is this employee at meeting deadlines for assigned tasks?"
  • "Rate the quality of this employee's work output during the evaluation period."
  • "How well does this employee collaborate with team members?"
Minimum and Maximum Values

The range of possible scores for this question. Values can be negative to allow for penalties in scoring.

Common ranges:
  • 0 to 10 (standard positive scoring)
  • -5 to 5 (allows for both penalties and rewards)
  • -10 to 10 (wider range for more nuanced evaluation)

Note: Negative scores can be used for performance issues that actually detract from an employee's overall contribution, like missed deadlines, quality problems, or policy violations.

Default Value

The pre-filled score that will appear when supervisors begin evaluations. This should represent a reasonable starting point for most employees.

Typical defaults:
  • For 0-10 range: 5 (middle of the range)
  • For -5 to 5 range: 0 (neutral starting point)
  • For complex metrics: A value that represents average performance
Weight Factor

Determines how much influence this question has on the final bonus calculation. Higher weights give the question more impact on the total score.

Weight examples:
  • 1.0: Standard weight (default)
  • 2.0: Double impact on final score
  • 0.5: Half impact on final score

Strategic use: Weights allow you to emphasize company priorities. For example, if quality is a critical focus area, questions about quality might have weights of 1.5 or 2.0.

Active Status

Controls whether this question appears in evaluations. Inactive questions are stored in the system but not presented to supervisors during evaluations.

This allows you to temporarily disable questions without deleting them, which is useful for:

  • Seasonal performance metrics
  • Questions that need revision
  • Maintaining historical data while updating evaluation criteria
